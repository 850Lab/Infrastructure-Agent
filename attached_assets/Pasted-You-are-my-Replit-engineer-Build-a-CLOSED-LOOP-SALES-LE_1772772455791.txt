PART 2 — OBSERVATION LAYER
==================================================

Create:
server/sales-learning/observation-engine.ts

Goal:
Every analyzed call should produce a structured observation record.

Inputs:
- existing transcript
- existing call record
- existing GPT deep analysis
- existing deterministic gatekeeper analysis

Observation extraction rules:
Capture these structured elements if present:
- opener used
- value prop used
- qualifying question count
- authority redirect attempted
- authority redirect success
- deflection phrase
- objection type
- engagement level
- outcome
- call duration
- gatekeeper name
- speaker mode

If speaker diarization is not available:
- set Detected_Speaker_Mode = Flat
- still extract best-effort observations
- do not block pipeline

Build helper functions:
- extractOpener()
- extractValueProp()
- countQualifyingQuestions()
- detectAuthorityRedirect()
- detectDeflectionPhrase()
- classifyObjectionType()
- classifyProspectEngagement()
- estimateTalkRatio()

Store all observation outputs in Call_Observations.

PART 4 — PATTERN LAYER
==================================================

Create:
server/sales-learning/pattern-engine.ts

Goal:
Learn across multiple calls, not just latest call.

Analyze patterns across:
- company
- gatekeeper
- operator
- industry category
- script version
- opener type
- objection type
- outcome type

Build these pattern aggregations:

1) Gatekeeper Patterns
Example:
"Company X gatekeeper used 'send it over' 3 times"

2) Script Performance Patterns
Example:
"Opener A yields 18% DM transfer vs Opener B yields 5%"

3) Objection Patterns
Example:
"Most common objection among tank cleaning contractors is 'already have cooling'"

4) Market Patterns
Example:
"Companies mentioning turnaround convert at 3x the rate"

5) Operator Patterns
Example:
"Operator tends to talk too long after first objection"

Rules:
- minimum sample size thresholds before creating Pattern_Insights
- assign Confidence_Score based on sample size + consistency
- create Recommended_Action fields

Segment keys examples:
- company:<id>
- gatekeeper:<name>
- category:tank_cleaning
- opener:heat_mitigation_infrastructure
- objection:already_have_cooling

PART 6 — PARALLEL OPERATION WITH EXISTING PLAYBOOKS
==================================================

Create:
server/sales-learning/run-sales-learning.ts

Command:
npx tsx server/sales-learning/run-sales-learning.ts --client=<client_id> --limit=50

Pipeline:
1) fetch recently analyzed calls not yet processed by sales learning
2) build Call_Observations
3) build Call_Learning
4) update Pattern_Insights
5) create/refresh Script_Patches

This must run in parallel with existing script generation.
It must NOT block playbook generation if it fails.

Add a flag to run-daily:
--salesLearning=true
Default true

If sales learning fails:
- log error
- continue rest of pipeline
- write error to Run_History
